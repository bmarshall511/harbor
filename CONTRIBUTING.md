# Contributing to Harbor

Thanks for contributing.

## Expectations
- Read `CLAUDE.md`, `AGENTS.md`, and relevant rules before making major changes.
- Discuss major architectural changes before implementing them.
- Keep code DRY, typed, tested, documented, and accessible.
- Preserve provider abstractions, Electron safety boundaries, and permission/audit guarantees.

## Development workflow
- install dependencies
- run the app locally
- run lint, typecheck, and tests before submitting changes
- update docs when behavior or architecture changes
